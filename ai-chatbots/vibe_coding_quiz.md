## Answer Key (Teacher Copy)

1. **B** — Describing what you want in plain English and using AI to help write the code. It's a collaborative workflow, not a replacement for understanding.

2. **B** — It's specific about what to build, what tech to use, what features to include, and has a constraint (one file). Specific prompts get dramatically better results.

3. **C** — Read through it and understand what each part does. If you can't explain the code, you haven't learned — you've just copied. Plus, AI code often has bugs you need to catch.

4. Example: "Build a trivia quiz app using HTML, CSS, and JavaScript. It should have at least 10 multiple-choice questions about [topic], show one question at a time with 4 answer options, track the user's score, and show a results screen at the end with the percentage correct. Keep it in a single HTML file with a modern, clean design." Key elements: specifies what (quiz), tech (HTML/CSS/JS), features (10 questions, score tracking, results), and constraint (single file).

5. This is wrong because you still need to understand code to: (1) write good prompts — you need to know what's possible, (2) evaluate if AI's code is correct — it often has bugs, (3) debug when things break, (4) make design decisions AI can't make for you, and (5) explain your code to others (teachers, employers, teammates). AI is a tool that makes you faster, not a replacement for understanding. It's like a calculator — powerful but useless if you don't understand math.
